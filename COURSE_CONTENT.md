## Course Overview

Better than REST API! Build a fast scalable HTTP/2 API for a Golang microservice with gRPC, Protocol Buffers (protobuf). Learn how gRPC works behind the scenes and implement various types of APIs.

### What You'll Learn
- Learn the gRPC theory to understand how gRPC works
- Compare gRPC and REST API paradigm
- Write your gRPC service definition in .proto files
- Generate Server & Client Code in Golang using the protoc gRPC Plugin
- Implement Unary, Server Streaming, Client Streaming & Bi-Directional Streaming API
- Practice your learning with Exercises & Solutions
- Implement advanced concepts such as Error Handling, Deadlines & SSL Security
- Implement a full CRUD API on top of MongoDB
- Get pointers to expand your learning journey and get inspired by real world gRPC services

### Prerequisites
- Good Understanding of Golang Programming Language
- Good Understanding of Protocol Buffers

## Key Concepts Learned

1. **gRPC Fundamentals** - Understanding the framework
2. **Protocol Buffers** - Service definition language
3. **HTTP/2** - Modern transport protocol
4. **Unary API** - Simple request-response
5. **Server Streaming** - Server sends multiple responses
6. **Client Streaming** - Client sends multiple requests
7. **Bi-Directional Streaming** - Both sides stream
8. **Error Handling** - gRPC error codes
9. **Deadlines** - Request timeouts
10. **SSL Security** - Encrypted communication
